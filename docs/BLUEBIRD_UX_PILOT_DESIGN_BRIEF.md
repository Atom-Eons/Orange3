# ORANGEBOX Bluebird UX Pilot Design Brief

Date: 2026-05-20

Use this document as the prompt/spec to pass into UX Pilot AI. UX Pilot is not a competitor in this context. UX Pilot is the design generation tool we will use to help visualize and iterate the full ORANGEBOX product experience.

This brief should push for the biggest, most polished version of the dream: a luxury AI creation and operations system for a non-coder operator, not a normal SaaS dashboard, not a dev console, and not a reduced chat app.

## Product Name And Structure

Top product surface: **AE See-Suite**

Operations surface: **AE Operations**

System name: **ORANGEBOX**

Networked AI-computer module: **Ethereal AI Link**

Avoid these product-facing words:

- cockpit
- BLUEB0X
- Codexa as a buyer-facing name
- settings as the main back-office label
- dev console language
- terminal-first framing

The product should feel like a premium creative command suite: part AI design studio, part executive project command center, part autonomous operations board, part proof-backed release system.

## Core Product Vision

AE See-Suite is where a non-coder creates with AI.

AE Operations is the back side of the same product: health, setup, proof, recovery, release, rollback, networked AI computer setup, and receipts.

The app should not feel like a regular chat app. The main problem it solves is that normal AI chat apps become long scrolling transcripts that hide the actual work. ORANGEBOX should make creation visible, structured, proof-backed, and recoverable.

The central experience:

1. The user describes what they want to build.
2. The system turns that into an operating route.
3. The route activates departments, macro-actions, proof gates, and artifacts.
4. Silent Canvas shows visible movement and mutation, not endless text.
5. The user can generate variants, compare directions, deepen a direction, and package the result.
6. AE Operations proves the system is healthy, installable, repairable, and releasable.

## Design Feeling

Aim for:

- luxury spatial software
- high-end AI creation suite
- calm executive control
- cinematic dark interface
- refined glass and light
- useful density without visual clutter
- premium motion
- strong first impression
- non-coder confidence

Do not design:

- a generic admin dashboard
- a CRUD app
- a terminal wrapper
- a flat SaaS analytics screen
- a developer-only app
- a landing page instead of a usable product

Visual language:

- OLED black / obsidian foundation
- orange energy as the ORANGEBOX signal
- emerald/cyan success and live-system accents
- soft slate text
- restrained premium glass panels
- glowing rails and fibers only where they communicate state
- 8px-or-less cards unless a true modal/artifact needs more
- no giant rounded marketing blobs
- no generic purple SaaS gradient
- no cartoon mascots

## Primary Layout Concept

The product can use a double-sided paper/card metaphor:

- Front side: **AE See-Suite** for creation.
- Back side: **AE Operations** for system truth, proof, setup, and recovery.

The flip should feel intentional and premium, like turning over a high-end mission sheet, not like a gimmick. It can be a spatial card flip, page turn, or command surface rotation.

Top navigation should be minimal:

- AE See-Suite
- AE Operations
- Projects
- Artifacts
- Receipts

There should not be a tiny settings gear as the main way to reach operations.

### 10. Advanced AI Computer Setup

Purpose: guide a buyer through controller + AI computer setup.

Required education:

- What is an AI computer?
- Why would I want one?
- Where can I buy one?
- What specs matter?
- Difference between internet network and direct AI link.
- Wi-Fi/router internet path vs direct Ethernet/Thunderbolt AI highway.

Required setup modules:

- Detect network adapters.
- Detect direct Ethernet possibility.
- Detect Thunderbolt possibility.
- Configure Ethereal AI Link.
- Test bandwidth.
- Optimize traffic priority.
- Save network profile.
- Show fallback tiers.

Buttons:

- Yes, Set Up AI Computer
- What Is an AI Computer?
- Check Ports
- Configure Direct Link
- Test Bandwidth
- Optimize Priority
- Save Profile
- Skip For Now

Interactions:

- The app should clearly say whether internet should stay on Wi-Fi/router while AI traffic uses direct cable.
- It should not promise RDMA/NVMe/TCP unless detected.
- It should support tiered paths:
  - direct Ethernet tuned TCP
  - SMB multichannel where available
  - future NVMe/TCP where OS supports it
  - Thunderbolt/USB networking where available
- It should block browser/game-launcher traffic from stealing the AI link where the OS allows safe policy.

## Global Interaction Rules

1. Every command creates or updates structured state.
2. Every meaningful mutation has visual feedback.
3. Every proof action writes a receipt.
4. Every cleanup or repair action shows exact targets first.
5. Every advanced hardware feature is detected before promised.
6. Every export previews included files.
7. Every generated page needs desktop and compact states.
8. Every stale product name is blocked from user-facing copy.
9. Every warning has a plain-English explanation and next action.
10. Every page should be usable by a non-coder.
11. Chat should never be the only memory of the work.
12. The system should always be recoverable after crash, restart, failed run, or stale helper process.

## Must-Have States

For every major screen, design:

- empty state
- loading state
- healthy state
- warning state
- failure state
- recovery state
- compact/mobile state
- proof/receipt state

## UX Pilot Generation Request

Generate a full high-fidelity UX system for ORANGEBOX Bluebird with all major pages above. Prioritize the main AE See-Suite creation experience, AE Operations final green board, Advanced AI Computer setup, and Silent Canvas. Make it feel premium, futuristic, and usable by a non-coder. Avoid generic dashboard layouts. Show real buttons, real cards, real states, and real navigation. Include desktop and compact variants where possible. The final design should feel like a complete creation operating system, not a partial concept.
