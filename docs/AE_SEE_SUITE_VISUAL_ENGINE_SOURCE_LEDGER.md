# AE See-Suite Visual Engine Source Ledger

Status: source-backed build ledger, not a completion claim.
Last updated: 2026-05-24

## Purpose

This ledger converts the operator visual direction, AE See-Suite PDFs, AIGPU protocol, Living Primitives spec, Silent Canvas market brief, and Grok upgrade material into concrete ORANGEBOX work. The product target is not a normal IDE or dashboard. It is a living spatial creation surface where the project is the nucleus and the work moves around it by relevance.

## Canonical Visual Target

- AE See-Suite is the front-side creation surface.
- AE Operations is the back-side proof, install, release, recovery, and health surface.
- Silent Canvas is the visual state engine, not a chat transcript.
- Chat is an ambient bottom portal, always available, never the main experience.
- AIGPU/Sonnet-style background work observes state, proposes HSMP mutations, and emits visual events; it never silently mutates state.
- The emotional target is slow, mellow, fun, chill, with earned awe.
- The interface should feel organic, bioluminescent, spatial, and calm, not a cold cyberpunk control panel.

## Current Correction

The previous AE See-Suite pass became too rectangular and dashboard-shaped. The current source-corrected pass changes the first surface to:

- organic field canvas behind the DOM;
- project nucleus drawn at the center;
- route, proof, mutation, artifacts, canvas, mission, and AIGPU phase drawn as orbiting relevance bodies;
- hard grid lines removed from the first screen;
- proof/route/brain text kept as soft peripheral organisms;
- command input kept at the bottom as an ambient portal.

## Non-Negotiables

- No fake living effects: every major visual state must be backed by route, workspace, proof, model, freeze, or operator input state.
- No fake green: screenshots and doctors prove the surface.
- No grid-first thinking on AE See-Suite front page.
- No vendor worship: Grok, Opus, GPT, Gemini, and local lanes are supported as model lanes, but ORANGEBOX owns the product behavior.
