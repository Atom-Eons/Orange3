# Orangebox Operating Law

Orangebox is the backend Ops system for local-first AI build work. It is not just an IDE, not a website, and not a store. It owns the control plane, receipts, gauntlets, local model lanes, worktree safety, deploy intake, and system proof.

## The Goal

The standing goal is a finished green Ops system:

```text
all active backend systems functional
all claims backed by receipts
rollback paths known
local machine state respected
no visual/website work unless explicitly authorized
no production deploy unless explicitly approved
```

## Core Roles

- Orangebox Ops backend owns execution, mutation, receipts, gauntlets, system proof, deploy intake, model/provider adapters, and rollback records.
- AECode is the middle voice/compiler contract for turning intent into bounded build outputs.
- Compression keeps work small: reuse, cartridge/registry, minimal hydration, local route, larger route only with warrant.
- Models are workers, not the brain. Deterministic runtime owns routing and proof.
