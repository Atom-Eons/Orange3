## The numbered ladder

These are your project's discrete work units. Edit freely.

1. **1A — Idea intake** — capture the original idea in one paragraph.
2. **1B — Project contract** — state goal, audience, acceptance criteria, hard constraints.
3. **1C — Source inventory** — list what files / repos / dependencies are in scope.
4. **1D — Memory recall** — read your own past lessons relevant to this project.
5. **1E — Scope map** — what's in, what's out, what's adjacent.
6. **1F — Risk map** — what could fail, what's protected, what needs approval.
7. **1G — Architecture route** — pick the technical approach.
8. **1H — Mission graph** — break into department-routable nodes.
9. **1I — UX and visual system** — only if there's a UI.
10. **1J — Implementation slice** — the first concrete buildable unit.
11. **1K — Local checks** — syntax, lint, types.
12. **1L — Visual proof loop** — screenshots, recordings, before/after.
13. **1M — Checkmate verification** — does it actually work?
14. **1N — Hallucination gate** — claims have receipts?
15. **1O — Security scan** — secrets, permissions, supply chain.
16. **1P — Review panel** — MIRRORS, ORANGE, LIPS pass.
17. **1Q — Release plan** — what does shipping look like?
18. **1R — Deploy / install smoke** — proof it works on a fresh system.
19. **1S — Receipt** — final receipt with rollback path.
20. **1T — Memory compile** — distill lessons back into Layer 2.

(You can shorten this. You can lengthen this. The shape is yours.)
