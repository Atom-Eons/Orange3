# PROJECT_SPINE — orangebox-onboarding

This is the **demo project** that ships with every new ORANGEBOX install. It walks you through ORANGEBOX by USING ORANGEBOX on the meta-project of "set up your operator cockpit."

You can rename it, edit it, or delete it. It exists so you see what a real project looks like instead of staring at an empty cockpit.

## Goal

Set up the operator's ORANGEBOX install so they can run real projects through it.

## Operator

You (filled in via your first-run profile, or "Operator" until you set one).

## Acceptance criteria

- ORANGEBOX is installed and launches cleanly.
- API key is configured (or explicitly skipped).
- Operator has seen the cockpit's command palette, DAG view, party-line, and receipts.
- Operator has either started their first real project OR archived this onboarding project.

## The numbered ladder

1. **1A — Idea intake** ✓ — capture why the operator bought ORANGEBOX.
2. **1B — Project contract** ✓ — state what "set up ORANGEBOX" means in one paragraph.
3. **1C — Source inventory** ✓ — installer + docs + 4-layer memory model.
4. **1D — Memory recall** ✓ — first-time install, no prior lessons yet.
5. **1E — Scope map** ✓ — what's in this onboarding project, what's adjacent.
6. **1F — Risk map** — what could fail on this operator's specific machine? (next action)
7. **1G — First real project** — operator creates their actual first project and switches active.
8. **1H — Archive this onboarding** — done, lessons compiled into knowledge vault.

## Why this demo project exists

Every new operator's cockpit boots into something real on day one. You're not configuring an empty dashboard — you're stepping into a working project that demonstrates the model. By the time you've watched the cockpit close out node 1F and dispatch 1G, you've seen the full ORANGEBOX loop without reading a manual.

When you're ready, rename this project to your real first project, or archive it and start fresh.
