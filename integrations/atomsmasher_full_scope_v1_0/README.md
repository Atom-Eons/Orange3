# AtomSmasher v1.0 — Full Scope Total Work Compiler

AtomSmasher v1.0 is a local Python + SQLite implementation of the full 620-addition AtomSmasher control plane.

Core law: **Only smart work is done.**

This package does not create 620 empty theatrical files. It registers all 620 additions as live executable capabilities and routes each one through concrete engine families: heat/order governance, full source ingest, commitment codec, equation memory, cache/runtime bridge, retrieval, proof/debt/evaluation, agent leases, code context, security, human attention, energy, memory lifecycle, and total work compilation.

## Run

```bash
python -m unittest discover -s tests -v
python -m atomsmasher --db demo.db v10-demo
python -m atomsmasher --db demo.db run-all-additions
python -m atomsmasher --db demo.db compile "continue AtomSmasher without losing orders"
```

## What is live

- All 620 additions are registered in SQLite.
- All 620 can be executed through `FeatureExecutor`.
- Every execution emits a receipt.
- Full text ingest stores raw source, chunks, FTS index, coverage receipt, extracted orders, atoms, and numeric equation packets.
- `orders:` and marching-order style text are HOT_ALWAYS mission law.
- Numeric series are represented as equations plus residuals where possible.
- Stable prompt prefixes are canonicalized for cache reuse.
- Exact/semantic cache, runtime profile receipts, least-action routing, expansion warrants, saved-work certificates, debt hooks, security scans, agent leases, and proof probes are implemented.

## Honest limitation

This is a working local control-plane implementation. It does not magically implement GPU-level KV cache mutation inside runtimes that are not present. Runtime/KV/LLM-specific additions are represented as executable capability profiles and adapter receipts today, ready to connect to vLLM/SGLang/llama.cpp/TensorRT-LLM when those runtimes are installed.

The important line is: **not scaffold; live software with receipts.** Some capabilities are native operational engines. Some are live adapter/profiling/control capabilities because the underlying external runtime may or may not exist on the user's machine.
