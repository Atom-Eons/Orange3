# Gremlin in the 5-Brain Stack

## Final stack

```text
Judgement 5.5 / Judgement Local
  ↑
Packets only
  ↑
Librarian  → context/files/tools
Forge      → build/code/commands
Mirror     → truth/tests/verification
Gremlin    → always-on unfiltered Misfit view
STRONGARM  → obedience/full-effort/anti-bullshit gate
```

## Three voices

```text
Builder voice:
  Librarian + Forge

Truth voice:
  Mirror

Unfiltered voice:
  Gremlin + STRONGARM
```

## Why Gremlin is separate from STRONGARM

STRONGARM is discipline.

Gremlin is edge.

STRONGARM asks:
```text
Did we obey the real request fully and lawfully?
```

Gremlin asks:
```text
What is the model pretending, hiding, softening, or refusing to say?
```

They are different.

Together:

```text
Gremlin breaks the fog.
STRONGARM forces the rewrite.
Mirror checks truth.
Judgement ships synthesis.
```

## Token economy

Gremlin is always-on but cheap.

It gets:

```text
request
draft or current plan
context digest
role mission
packet schema
```

It does not get:

```text
full chat history
all files
all model debates
all source text
```

That keeps the unfiltered view present without burning the machine.
