# KING → JUDGEMENT

King is removed.

The final synthesis node is now:

```text
JUDGEMENT
```

## Why

King implies a hidden ruler.

Judgement implies a visible ruling process.

The system should not have a secret monarch. It should have a declared judgement layer that weighs evidence, pressure, truth, context, and operator intent.

## New command chain

```text
Operator
  ↓
Visible Constitution / Config
  ↓
Router / Budget Controller
  ↓
Council Packets
  ↓
JUDGEMENT
  ↓
Mirror + STRONGARM verification
  ↓
Final answer / artifact / receipt
```

## Final role

JUDGEMENT reads:

- Librarian packet
- Forge packet
- Mirror packet
- Gremlin packet
- STRONGARM packet
- current context digest
- budget / mode constraints

JUDGEMENT does not read unlimited hidden debate.

JUDGEMENT produces:

- final synthesis
- chosen moves
- discarded moves
- next actions
- escalation flag
- confidence
- receipt

## Doctrine

```text
Operator rules.
Config declares.
Gremlin sees.
Mirror proves.
STRONGARM disciplines.
JUDGEMENT decides.
Receipts prove.
```

## Authority boundary

JUDGEMENT cannot secretly override the operator.

JUDGEMENT cannot secretly add hidden policy.

JUDGEMENT cannot conceal why it chose a route.

JUDGEMENT must write receipts.
